# 交叉审阅

你将审阅另一个 Agent 的方案，并提供专业的评审意见。

## 对方方案
详见 `{{PROPOSAL_FILE}}`，请先阅读该文件。

## 审阅要点
- 方案的完整性和可行性
- 潜在的技术风险和遗漏
- 与你的方案的异同之处
- 建设性的改进建议

## 你的方案（供参考）
你之前的方案已保存在 `{{MY_PROPOSAL_FILE}}`，可对比参考。

## 重要约束
- 将审阅结果输出到 `{{REVIEW_RESULT_FILE}}`
- 审阅结果应包含：
  1. **总体评价**：对方方案的优缺点
  2. **关键分歧**：你不同意的地方及理由
  3. **共识点**：双方一致的部分
  4. **改进建议**：具体、可操作的建议
- **完成后创建信号文件** `{{SIGNAL_FILE}}`，JSON 格式如下：

```json
{
  "status": "done",
  "brain_id": "<从 artifact 路径中提取你的 conversation-id>",
  "branch": "",
  "files_changed": [],
  "summary": "一句话描述审阅结论",
  "skills_used": ["列出你读取和使用的 skill 名称"],
  "errors": []
}
```

> 💡 交叉审阅不需要创建/切换分支，`branch` 字段留空。

## 项目上下文
详见 `{{CONTEXT_FILE}}`，请先阅读该文件获取项目背景。

> ⚠️ **如果任务失败**，仍然必须创建信号文件，但将 `status` 改为 `"failed"`，并在 `errors` 中描述原因。
